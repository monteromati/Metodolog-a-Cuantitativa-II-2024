Evaluaci√≥n de los Supuestos del Modelo de Regresi√≥n

Es fundamental evaluar los supuestos estad√≠sticos del modelo de regresi√≥n para asegurarnos de que los resultados sean v√°lidos.

  • Supuesto de Linealidad El supuesto de linealidad indica que la relaci√≥n entre el predictor (ses) y la variable dependiente (math) es lineal. Para evaluar la linealidad, podemos utilizar un diagrama de dispersi√≥n (scatter plot) que muestre la relaci√≥n entre la variable predictora ses (√çndice de Estatus Econ√≥mico, Social y Cultural) y la variable dependiente math (Puntaje en Matem√°ticas).
ggplot(pisa2009_chl) +
  aes(x = ses, y = math) +
  geom_point(colour = "#0c4c8a") +
  theme_minimal()
Warning: Removed 89 rows containing missing values or values outside the scale range
(`geom_point()`).

Interpretaci√≥n: Si los puntos en el gr√°fico muestran una tendencia a alinearse en una l√≠nea recta (ascendente o descendente), podemos asumir que la relaci√≥n es aproximadamente lineal. Si los puntos siguen una forma curvil√≠nea, la relaci√≥n podr√≠a no ser lineal, lo que sugerir√≠a la necesidad de transformar las variables o considerar un modelo diferente.

  • Supuesto de Normalidad de los Residuos El supuesto de normalidad de los residuos indica que los errores del modelo (es decir, las diferencias entre los valores observados y los valores predichos) deben distribuirse normalmente. Esto es crucial para validar las pruebas de significancia estad√≠stica (prueba t de Student) que se utiliza en la regresi√≥n. Para evaluar este supuesto, utilizamos varias aproximaciones:

    • Histograma de los residuos: Permite observar la distribuci√≥n de los errores.
    • Gr√°fico de densidad: Proporciona una visualizaci√≥n suave de la distribuci√≥n de los residuos.
    • Gr√°fico Cuantil-Cuantil (Q-Q plot): Compara la distribuci√≥n de los residuos con una distribuci√≥n normal te√≥rica.
    • Prueba de Shapiro-Wilk: Prueba estad√≠stica que eval√∫a la normalidad de los residuos.
hist(m1$residuals, freq = FALSE)
plot(density(m1$residuals))
qqnorm(m1$residuals, main = 'Gr√°fico Cuantil-Cuantil')
qqline(m1$residuals)
shapiro.test(m1$residuals[0:5000])

  • Supuesto de Homocedasticidad El supuesto de homocedasticidad establece que la varianza de los residuos es constante a lo largo de todos los valores de la variable predictora. Esto significa que los errores del modelo no deber√≠an mostrar patrones sistem√°ticos cuando se grafican contra los valores predichos. La falta de homocedasticidad, conocida como heterocedasticidad, puede llevar a estimaciones sesgadas de los coeficientes de regresi√≥n.

Para evaluar este supuesto, utilizamos el test de Breusch-Pagan, que es una prueba estad√≠stica que eval√∫a la presencia de heterocedasticidad. Si el test de Breusch-Pagan indica Un valor p > .05, entonces este sugiere que no se puede rechazar la hip√≥tesis nula de homocedasticidad, indicando que la varianza de los residuos es constante. Si el valor p es menor a 0.05, hay evidencia de heterocedasticidad, lo que podr√≠a requerir la transformaci√≥n de variables o el uso de un modelo robusto.

bptest(math ~ ses, data = pisa2009_chl)

  • Evaluaci√≥n de la Multicolinealidad En este caso como tenemos 2 o m√°s predictores, debemos evaluar la multicolinealidad en el modelo. La multicolinealidad ocurre cuando dos o m√°s variables predictoras en un modelo de regresi√≥n est√°n altamente correlacionadas entre s√≠. Esto puede ser problem√°tico porque impide la estimaci√≥n de un modelo de regresi√≥n. Para ello, observamos las correlaciones entre los predictores y calculamos el factor de inflaci√≥n de la varianza (VIF).

Podemos evaluar la presencia de multicolinealidad en un modelo de regresi√≥n generando una matriz de correlaci√≥n de los predictores puede dar una idea inicial sobre las relaciones bivariadas entre ellos. Observamos si hay correlaciones altas (generalmente r > 0.8 o < -0.8) entre dos variables predictoras pueden ser una indicador de multicolinealidad.

data <- pisa2009_chl[, c("math", "ses", "read", "scie")]
correlaciones <- cor(data, use = "complete.obs")

# Visualizar la matriz de correlaci√≥n usando un correlograma
corrplot(correlaciones, method = 'circle', type = 'lower', insig = 'blank',
         addCoef.col = 'black', number.cex = 0.8, order = 'AOE', diag = FALSE)

Tambi√©n, podemos estimar el factor de inflaci√≥n de la varianza (VIF). El VIF mide cu√°nto la varianza de un coeficiente de regresi√≥n est√° inflada debido a la multicolinealidad. Un VIF de 1 indica que no hay correlaci√≥n entre la variable predictora y las dem√°s variables predictoras. Un VIF entre 1 y 5 sugiere una correlaci√≥n moderada pero aceptable. Un VIF mayor que 5 (algunos usan el umbral de 10) indica una alta multicolinealidad que podr√≠a ser problem√°tica.

kable(vif(m2), digits = 2)
x
ses 1.24
female 1.13
read 3.94
scie 3.77

Interpretaci√≥n: Si alguna de las variables en el modelo tiene un VIF alto (por ejemplo, mayor a 5), esto indica que la variable est√° altamente correlacionada con una o m√°s de las otras variables predictoras, lo que sugiere multicolinealidad.

¬øQu√© hacer si hay multicolinealidad? - Eliminar una de las variables correlacionadas: Si dos variables est√°n altamente correlacionadas, podr√≠as considerar eliminar una de ellas para reducir la multicolinealidad. - Transformar las variables: En algunos casos, la transformaci√≥n de las variables (por ejemplo, usando logaritmos o diferencias) puede reducir la multicolinealidad.

Esta funci√≥n obtiene gr√°ficos de diagn√≥stico para chequear supuestos.

check_model(m1)
